# Ark Coach reference material

The Camtek User Manual grounds the Ark Coach in product specifics. Because
the PDF is too large to inline on every Claude API call (Anthropic enforces a
~32 MiB per-request body limit), we **index it once** into a small searchable
text file, then retrieve the most relevant chunks per call.

## One-time setup

1. Drop the PDF here as `camtek-user-manual.pdf`, or set `USER_MANUAL_PATH`
   in `.env` to an absolute path elsewhere (e.g. `C:\Ark\data\camtek-user-manual.pdf`
   on the production VM, so the manual survives redeploys).
2. Run the indexer from `src/backend`:
   ```bash
   npm run index-manual
   ```
   It walks the PDF page-by-page, extracts text, and writes
   `manual-index.json` next to this README. A 500-page manual takes a minute or two.
3. Restart the backend. Startup log should read:
   `Manual index loaded from …: N chunks across N pages.`

If `manual-index.json` is missing or unreadable, the Ark Coach runs without
product context (the backend logs a single warning at startup).

## Runtime behaviour

Each Claude API call (`/api/ai/chat`, `/api/ai/suggest`, `/api/azure/attachments/scan`)
builds a query from the active draft state, picks the top-4 relevant chunks
via in-memory keyword search, and prepends those chunks (as text) to the
system prompt. The original PDF is **never** sent to Anthropic.

Override the index file location with `USER_MANUAL_INDEX_PATH` if needed.

## Refreshing after a manual update

When the manual changes, re-run `npm run index-manual` and restart the
backend. The index file records the source PDF's SHA-256 so drift can be
detected later if we want to.

## v2 (planned)

Image captioning via Claude Vision: for pages with figures and screenshots,
emit a 2-3 sentence caption alongside the page text so visual content is
searchable. The current index file format already carries `imagePaths` per
chunk for this future enhancement.
